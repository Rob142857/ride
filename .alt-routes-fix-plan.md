# Alternative Routes Bug Fix Plan

## Root Issues Found

### 1. saveRouteData crashes on [undefined] alternative route names
- `routeNames` contains `[undefined]` when `route.summary?.name` is undefined
- Object.entries(undefined) throws TypeError
- FIX: fallback to `route.name || route.label || \`Route ${idx+1}\``

### 2. saveRouteData doesn't persist alternatives to DB
- It sets `this.currentTrip.route_metadata` but never calls `API.trips.saveAltRoutes`
- So alternatives are lost on refresh
- FIX: add API call after saving to `currentTrip`

### 3. Dual route panels showing on mobile (no CSS applied)
- `renderMobileRouteInfo()` returns dual panels, both visible simultaneously
- `route-alternatives` CSS expects `.visible` / `.hidden` classes but none are ever applied
- FIX: use CSS classes on container, only show correct panel

### 4. UI Controller _showRouteChoicePanel click handler broken
- Creates `routeOptionClickHandler` but never attaches it to any element
- The `button.addEventListener('click', routeOptionClickHandler)` uses `routeOptionClickHandler` as a bare variable, but the handler only exists inside the loop closure
- It should bind a separate click handler per alternate route that calls `this.setRouteAlternative`
- FIX: create closure or use data-alt-index attribute + event delegation

### 5. updateRoute overwrites saved alternatives immediately
- Calls `await this.saveRouteData(route, origin, destination, alternatives)` to save alt routes
- Then immediately does `this.currentTrip.route = { coordinates, distance, duration, steps }`
- This wipes out the metadata the previous function just set
- FIX: preserve alternativeRoutes metadata when merging route data

### 6. getSelectedMapLineCoordinates uses wrong property name
- Looks for `this.currentTrip.alternativeRoutes` (camelCase, never set by `loadTripData`)
- Should use `this.currentTrip.alternative_routes` or normalize property names
- FIX: normalize property names on load

### 7. _showRouteChoicesOverlay wrong property name
- Reads `altRoute.route_index` which is never set
- Should use `idx` from iteration
- FIX: use `idx` directly

### 8. No panel cleanup on trip load
- `_showRouteChoicePanel` only clears inside the overlay, but old panel may persist
- Overlays may pile up after multiple route queries
- FIX: clear previous overlays before creating new ones

### 9. renderPanel never called from anywhere
- `RouteAlternatives.renderPanel()` returns DOM but nothing consumes it
- It's a dead module
- FIX: remove dead code or integrate into renderMobileRouteInfo

### 10. loadTripData clears alternative_routes
- Clears `this.currentTrip.alternative_routes = []` then never restores from backup
- Makes it impossible to access saved alt routes after load
- FIX: restore from trip property

## Fix Order
1. Fix property normalization in loadTripData + saveRouteData
2. Fix saveRouteData to persist to DB
3. Fix updateRoute to not overwrite alt route data
4. Fix _showRouteChoicePanel click handling
5. Fix _applyActiveRouteSelection
6. Fix renderMobileRouteInfo to use CSS classes
7. Fix getSelectedMapLineCoordinates and _showRouteChoicesOverlay property names
